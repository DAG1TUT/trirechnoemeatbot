# Telegram-бот учёта смен и ежедневной отчётности

MVP для сети мясных торговых точек: продавцы открывают/закрывают смены и отправляют отчёты через бота, руководитель видит активные смены и получает итоговый отчёт за день.

## Стек

- Python 3.10+
- aiogram 3.x
- SQLAlchemy 2.x (async, aiosqlite)
- SQLite (архитектура готова к переходу на PostgreSQL)
- FSM для пошагового ввода отчёта
- .env для конфигурации

## Архитектура

- **Роли**: гость (привязка к продавцу), продавец, админ. Роль определяется в middleware по `telegram_id` (таблицы `admins` и `sellers` + опционально `ADMIN_IDS` из .env).
- **Бизнес-логика** в `services/`: открытие/закрытие смен с проверками (одна смена у продавца, одна точка не занята двумя), сборка отчёта, напоминания.
- **Данные** в `repositories/`, модели в `core/models/`. Сессия БД создаётся в middleware на каждый апдейт и коммитится после успешного ответа.
- **Итоговый отчёт** за день отправляется админам один раз при закрытии последней смены (таблица `daily_report_status`). Руководитель может вручную запросить «Итоговый отчёт» и «Отчёт за сегодня» в любой момент.
- **История** — отчёт по введённой дате (кнопка «История» → ввод ДД.ММ.ГГГГ).
- **Напоминания**: сервис `reminder_service` (незакрытые смены/точки) готов к вызову по расписанию (APScheduler/cron).

## Запуск

1. Клонировать репозиторий и перейти в каталог проекта.

2. Создать виртуальное окружение и установить зависимости:

   ```bash
   python -m venv venv
   source venv/bin/activate   # Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. Создать файл `.env` (скопировать из `.env.example`):

   ```bash
   cp .env.example .env
   ```

4. В `.env` указать:
   - `BOT_TOKEN` — токен от @BotFather.
   - При необходимости `ADMIN_IDS` — telegram_id руководителя через запятую (например `123456789`). Эти пользователи будут считаться админами даже без записи в таблице `admins`.

5. Инициализировать БД и загрузить seed-данные:

   ```bash
   python -m scripts.seed
   ```

   Будут созданы: 4 тестовых продавца (Продавец 1–4), 4 торговые точки по адресам, один админ с `telegram_id=0`. Чтобы руководитель получил доступ, либо укажите его в `ADMIN_IDS` в `.env`, либо после первого запуска обновите в БД запись в таблице `admins` (поле `telegram_id`).

6. Запустить бота:

   ```bash
   python run.py
   ```
