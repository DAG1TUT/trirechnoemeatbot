# Деплой бота на Railway

## 1. Постоянная база (чтобы данные не пропадали после деплоя)

Без PostgreSQL при каждом деплое контейнер пересоздаётся и все данные (отчёты, продавцы, точки) теряются. Чтобы данные хранились всегда:

1. В проекте Railway нажми **New** → **Database** → **Add PostgreSQL** (или в каталоге проекта кнопка **+ New** → **Database** → **PostgreSQL**).
2. Дождись создания БД. Открой сервис PostgreSQL.
3. Во вкладке **Variables** или **Connect** скопируй переменную **`DATABASE_URL`** (или **`DATABASE_PRIVATE_URL`** — она для доступа изнутри Railway).
4. Открой сервис **твоего бота** (не БД). В **Variables** добавь переменную:
   - **Name:** `DATABASE_URL`
   - **Value:** вставь скопированный URL (формат `postgresql://user:pass@host:port/railway` — бот сам подставит драйвер `asyncpg`).
5. Сохрани переменные. Railway перезапустит бот — он подключится к PostgreSQL, создаст таблицы, при первом запуске выполнится seed (продавцы, точки, админ). При следующих деплоях данные уже будут в БД и никуда не денутся.

Если база уже была (например, раньше использовали SQLite в контейнере), то после перехода на PostgreSQL таблицы будут пустые — один раз снова отработает seed. Старые отчёты из SQLite перенести можно только вручную (экспорт/импорт).

## 2. Переменные окружения бота

В настройках **сервиса бота** (Variables) должны быть:

| Переменная     | Описание                              | Пример              |
|----------------|----------------------------------------|---------------------|
| `BOT_TOKEN`    | Токен бота от @BotFather               | `123:AAH...`        |
| `ADMIN_IDS`    | Telegram ID руководителя (через запятую) | `8194223689`      |
| `DATABASE_URL` | URL от PostgreSQL (см. выше)           | из сервиса PostgreSQL |
| `ADMIN_PASSWORD` | Пароль кнопки «Администратор» **в Telegram-боте** | отдельно от веб-админки |

**Веб-дашборд (админ-панель на сайте):**

| Переменная | Описание |
|------------|----------|
| `DASHBOARD_ADMIN_PASSWORD` | Пароль **только для входа на сайте** (`/admin/login`). **Задай отдельно** от `ADMIN_PASSWORD`. Если не задан — для старых проектов подставляется `ADMIN_PASSWORD`. |

**Веб-кабинет продавца** (дополнительно):

| Переменная | Описание |
|------------|----------|
| `SELLER_WEB_SESSION_SECRET` | Случайная строка для подписи cookie-сессии входа продавца на сайте. **Задай в продакшене** (иначе используется небезопасный дефолт из кода). |

Если база **PostgreSQL уже существовала** до появления веб-кабинета, один раз выполни в SQL (Query в Railway PostgreSQL или `psql`):

```sql
ALTER TABLE sellers ADD COLUMN IF NOT EXISTS web_password_hash VARCHAR(255);
```

Файл с тем же текстом: `scripts/migrations/add_sellers_web_password_hash.sql`.  
Подробности и проверка: **`scripts/migrations/README.md`**. С локального компьютера (если установлен `psql`): `bash scripts/migrations/apply_web_password_column.sh` при заданном `DATABASE_URL`.

- Стартовая страница (выбор роли): `https://<твой-хост>/`
- Вход в админ-панель (пароль): `/admin/login` → затем дашборд `/dashboard`
- Дашборд руководителя (после входа): `/dashboard`
- Вход продавца: `/seller-cabinet/login`, регистрация: `/seller-cabinet/register` (только продавцы **без** веб-пароля в БД)

## 3. После деплоя

1. Открой бота в Telegram и отправь `/start`.
2. С аккаунта из `ADMIN_IDS` отправь боту `/start` — откроется меню руководителя, на этот чат будут приходить отчёты.

Готово: отчёты и все данные хранятся в PostgreSQL и сохраняются после каждого нового деплоя.
