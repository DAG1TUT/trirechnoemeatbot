## Что это

Telegram-бот для учёта личных расходов. Пользователь пишет или говорит голосом «на что и сколько» потратил — бот сохраняет трату и сам относит её к категории (еда, транспорт, машина, кафе и т.д.). Есть кнопки меню, статистика по категориям, голосовой ввод.

---

## Стек и файлы

- **Python 3.9+**, зависимости в `requirements.txt`: python-telegram-bot, SpeechRecognition, pydub.
- **bot.py** — точка входа, обработчики команд и сообщений, парсинг трат, голос, кнопки (Reply + Inline), логика «маленькая сумма + заправка/машина = умножить на 1000».
- **database.py** — SQLite (`expenses.db`): добавление трат, выборка по пользователю/категории/периоду.
- **categories.py** — словарь категорий и ключевых слов; `detect_category(description)`; порядок категорий важен (первое совпадение).
- **Procfile** — для деплоя: `worker: python bot.py`.
- **.gitignore** — исключены venv/, expenses.db, .env. Токен бота не коммитить.

## Деплой и окружение

- **GitHub:** репозиторий `DAG1TUT/elegram-expense-bot` (ветка main).
- **Railway (Hobby):** сервис подтягивает код из GitHub, автодеплой при push. Переменная окружения: `TELEGRAM_BOT_TOKEN`. Бот работает 24/7 на сервере.
- **Локально (Mac):** папка `telegram-expense-bot`, запуск через `./venv/bin/python bot.py` с `TELEGRAM_BOT_TOKEN` в окружении или `./run.sh`. Для голоса нужен ffmpeg (brew install ffmpeg); на Railway голос может не работать (нет ffmpeg).

## Важные детали реализации

- Парсинг трат: число + описание из текста; русские числительные («две с половиной тысячи», «пятьсот») заменяются в `_replace_russian_amounts()`; суммы 1–10 при словах типа «заправка/машина» умножаются на 1000.
- Категория «Машина / Авто» идёт выше «Транспорт», заправка/бензин/машина относятся к авто.
- Голос: скачивание ogg с Telegram → конвертация в wav (pydub, ffprobe) → recognize_google(ru-RU). На Mac в коде подставляются пути Homebrew для ffmpeg/ffprobe.
- Кнопки: Reply-клавиатура (Общие траты, По категориям, Последние траты) + Inline по категориям; список категорий для кнопок берётся из `get_all_categories()` (все категории из categories.py), суммы из БД.
